# 数据获取与缓存

回测的第一步是拿数据。QKA 帮你搞定下载、缓存、合并，你只需要告诉它要哪些股票。

---

## 一句话用法

```python
data = Data(symbols=['000001.SZ', '600000.SH'])
```

就这一句，它会：

1. 从 baostock 下载日线数据
2. 每只股票存一个 parquet 文件到本地
3. 下次再跑，直接读缓存，不用重下

---

## 参数说明

| 参数 | 作用 | 默认值 |
|------|------|--------|
| `symbols` | 股票代码列表 | 必填 |
| `period` | 数据周期，日线填 `'1d'` | `'1d'` |
| `adjust` | 复权方式：`'qfq'`（前复权）、`'hfq'`（后复权）、`'bfq'`（不复权） | `'qfq'` |
| `source` | 数据源：`'baostock'` 或 `'akshare'` | `'baostock'` |

```python
# 最近一年，后复权
data = Data(symbols=['000001.SZ'], period='1d', adjust='hfq')
```

---

## 数据存在哪

```
项目目录/
└── datadir/
    └── baostock/
        └── 1d/
            └── qfq/
                ├── 000001.SZ.parquet
                ├── 600000.SH.parquet
                └── ...
```

每只股票一个文件，互不干扰。想清缓存，删掉这个文件夹就行。

---

## 合并后的数据长什么样

多个股票的回测里，QKA 会把它们拼成一张大表。列名格式是 `{股票代码}_{字段名}`：

| 日期 | 000001.SZ_close | 000001.SZ_volume | 600000.SH_close | 600000.SH_volume |
|------|----------------|------------------|----------------|------------------|
| 2024-01-02 | 10.2 | 50000 | 8.1 | 30000 |
| 2024-01-03 | 10.5 | 45000 | 8.3 | 28000 |

策略里取数据不用管这个格式，`self.get('close')` 帮你自动拆好了。

## 两种加载模式

### 普通模式（默认）

直接用。小数据量（几十只股票，一两年）没问题。

```python
data = Data(symbols=['000001.SZ'])
```

### 懒加载模式（lazy）

数据量大时用。比如 300 只股票跑 5 年，普通模式可能一次加载太久。懒加载用 dask 分批计算，内存峰值低很多。

```python
data = Data(symbols=[...很多股票...])
# 回测时传 lazy=True
bt.run(lazy=True)
```

怎么选：

| 情况 | 推荐 |
|------|------|
| 几个股票测个思路 | 普通模式，省事 |
| 几十上百个股票 | 懒加载，跑得快 |

两种模式下策略代码不用改，`self.get()` 和 `self.history()` 用法完全一样。

---

## 数据源怎么选

| 数据源 | 优点 | 缺点 |
|--------|------|------|
| **baostock**（默认） | 稳定，不用注册，开箱即用 | 只能单线程下载（它的限制） |
| **akshare** | 支持并发，速度快 | 个股接口偶尔被服务器屏蔽（不是 akshare 的问题） |

```python
# 用 akshare
data = Data(symbols=['000001.SZ'], source='akshare')
```

一般用 baostock 就够了。akshare 在 baostock 出问题时作为备选。
