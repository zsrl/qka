# 数据

回测的第一步是拿数据。QKA 帮你搞定下载、缓存、合并，你只需要告诉它要哪些股票。

---

## 一句话

```python
data = Data(symbols=['000001.SZ', '600000.SH'])
```

就这一句，它会：

1. 从 baostock 下载日线数据
2. 每只股票存一个 parquet 文件到本地
3. 下次再跑，直接读缓存，不用重下

## 参数

| 参数 | 作用 | 默认值 |
|------|------|--------|
| `symbols` | 股票代码列表 | 必填 |
| `period` | 数据周期 | `'1d'` |
| `adjust` | 复权方式：`'qfq'`（前复权）、`'hfq'`（后复权）、`'bfq'`（不复权） | `'qfq'` |
| `source` | 数据源：`'baostock'` 或 `'akshare'` | `'baostock'` |

```python
data = Data(symbols=['000001.SZ'], adjust='hfq')
```

## 数据存在哪

```
datadir/
└── baostock/
    └── 1d/
        └── qfq/
            ├── 000001.SZ.parquet
            ├── 600000.SH.parquet
            └── ...
```

每只股票一个文件。想清缓存删掉这个文件夹就行。

## 合并后的数据

多个股票回测时，QKA 拼成一张大表。列名格式是 `{股票代码}_{字段名}`：

| 日期 | 000001.SZ_close | 600000.SH_close |
|------|----------------|----------------|
| 2024-01-02 | 10.2 | 8.1 |
| 2024-01-03 | 10.5 | 8.3 |

策略里不用管这个格式，`self.get('close')` 自动帮你拆好。

## 两种加载模式

**普通模式** — 小数据量（几十只股票）直接用：

```python
data = Data(symbols=['000001.SZ'])
```

**懒加载（lazy）** — 几百只股票时用，分批计算，内存更低：

```python
bt.run(lazy=True)
```

策略代码不用改，两种模式 `self.get()` 和 `self.history()` 用法一样。

## 数据源

| 数据源 | 优点 | 缺点 |
|--------|------|------|
| **baostock**（默认） | 稳定，开箱即用 | 单线程下载 |
| **akshare** | 支持并发 | 个股接口偶尔被屏蔽 |

```python
data = Data(symbols=['000001.SZ'], source='akshare')
```
